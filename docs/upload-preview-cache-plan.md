# 上传预览缓存改进记录

## 背景

素材库已支持七牛和缤纷云上传进度展示。七牛私有空间图片预览可以通过签名 URL 正常访问；七牛私有视频远程流式播放可能初始化失败，因此当前实现会在失败后用签名 URL 下载到临时文件，再用本地文件播放器兜底播放。

这个兜底能解决“私有视频无法播放”，但如果每次点击预览都重新下载大视频，会带来等待时间和流量问题。后续需要补一层稳定缓存。

## 当前缓存现状

- 列表图片缩略图使用 `CachedNetworkImage`，已有磁盘缓存。
- 预览大图使用 `PhotoView(NetworkImage(url))`，主要依赖内存缓存，不是稳定磁盘缓存。
- 列表视频缩略图使用 ffmpeg 抽首帧并按 URL hash 缓存到本地。
- 视频预览优先远程流式播放；远程初始化失败时下载到临时文件兜底，但临时文件目前不按素材复用。
- 七牛私有 URL 会带 `e` 和 `token`，如果缓存 key 使用完整 URL，签名变化会导致缓存命中率变差。

## 后续建议

### 1. 图片预览改用稳定磁盘缓存

预览大图不要直接使用 `NetworkImage(url)`，建议改为 `CachedNetworkImageProvider` 或同等缓存方案。

私有空间不要用完整签名 URL 作为缓存 key，应使用稳定 key，例如：

```text
qiniu:<objectKey>
bitiful:<bucket>:<objectKey>
```

这样签名 URL 刷新后，仍能命中同一份图片缓存。

### 2. 视频兜底下载做素材级缓存

七牛私有视频远程流式播放失败后，下载到本地时不要只写临时文件。建议按素材生成稳定缓存路径：

```text
preview-video/<storageProvider>/<hash(bucket + objectKey + fileSizeBytes)>.mp4
```

流程建议：

1. 先尝试远程流式播放签名 URL。
2. 远程初始化失败时，检查本地预览缓存文件是否存在。
3. 存在则直接用 `VideoPlayerController.file` 播放。
4. 不存在才下载签名 URL 到缓存文件。
5. 下载完成后播放本地文件。

这个缓存只用于远程播放失败的兜底，不改变公开视频和缤纷云正常流式播放路径。

### 3. 给视频兜底下载展示准备进度

大视频兜底下载时需要明确 UI 状态，避免用户以为卡死。

建议预览播放器增加状态：

```text
正在准备预览
正在下载 37%
预览准备失败
```

下载进度可以复用 `AppState.downloadRemoteVideoToTemp(... onProgress)` 的进度回调。

### 4. 增加缓存清理策略

视频预览缓存不能无限增长。建议做一个简单上限策略：

- 总大小上限：例如 1GB。
- 数量上限：例如最近 20 个视频。
- 删除策略：按最后访问时间从旧到新清理。

缓存命中时更新最后访问时间；素材删除时可以顺手删除对应预览缓存。

## 推荐优先级

优先做第 2 和第 3 点：七牛私有视频兜底下载缓存 + 下载进度 UI。

这两项直接解决“大文件反复点击预览会卡、会重复下载”的问题，风险也最小。图片预览缓存和全局清理策略可以随后补齐。
